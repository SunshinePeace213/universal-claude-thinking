**As a** Claude Code,
**I want** a lightweight main CLAUDE.md file that orchestrates module loading,

6. Ensure compatibility with Claude Code's native @import syntax

[Source: architecture/backend-architecture.md#module-loading]

- **Technology**: Claude Code's native `@import` mechanism
- **Import Syntax**: `@import ".claude/thinking-modules/{{module}}.md"`
- **Security**: SHA-256 validation before loading (from Story 1.8)
- **Path Restrictions**: Imports restricted to `.claude/` directory only
- **Performance Targets**:
  - Classification: <100ms
  - Module Load: <50ms per module

### Request Classification System

[Source: architecture/data-models.md#request-classification]
Classification categories and module mapping:

```typescript
interface RequestClassification {
  category: 'simple' | 'complex' | 'search' | 'code' | 'meta';
  confidence: number;
  requiredModules: string[];
  suggestedAgents: string[];
  mcpTools: string[];
  estimatedTokens: number;
}
```

Category to Module Mapping:

- **simple**: ['response-formats']
- **complex**: ['SAGE', 'SEIQF', 'cognitive-tools/analysis']
- **search**: ['SIA', 'cognitive-tools/search']
- **code**: ['SEIQF', 'cognitive-tools/code-analysis']
- **meta**: ['SAGE', 'cognitive-tools/meta-reasoning']

### Claude Code Import Syntax

[Source: architecture/coding-standards.md#import-conventions]

- Always use relative paths from `.claude/` directory
- Import pattern: `@import "./relative/path/to/module.md"`
- No dynamic string concatenation in imports
- Imports must be statically analyzable

### Debug Header Requirements

[Source: architecture/backend-architecture.md#visibility-logging]
Debug header must include:

- Active modules list with emoji indicators
- Total token count
- Module load sequence and timing
- Clear-thought MCP tool status
- Keep under 200 tokens total

Example format:

```
🎯 Active Modules: SAGE (2.1K), response-formats (1K)
⚡ Classification: complex (0.92 confidence)
📊 Total Tokens: 3,100 / 5,000
```

### Technical Constraints

- No eval() or dynamic code execution allowed
- All paths must be validated and sanitized
- Module loading must gracefully handle failures
- Classification must complete within 100ms
- Total orchestrator file must stay under 500 tokens
