# Story 1.3: Basic Request Classifier

## Status

Completed

## Story

**As a** system,
**I want** to classify incoming requests to determine which thinking modules to activate,
**so that** context window usage is minimized while maintaining functionality.

## Acceptance Criteria

1. Classifier completes analysis within 100ms
2. Support classification into 5 main categories (simple query, complex analysis, search-based, code generation, meta-reasoning)
3. Return list of recommended modules with priority scores
4. Log classification decisions for future optimization
5. Provide confidence score for classification decisions
6. Handle edge cases where classification is ambiguous

#### Areas for Improvement

1. **Code Quality Enhancements**

   **Finding**: Magic numbers throughout the code
   - Lines 163, 170, 187, 194: Hard-coded confidence adjustments (0.3, 0.4)
   - Line 215: Hard-coded fallback confidence (0.8)
   
   **Recommendation**: Extract to named constants
   ```javascript
   const CONFIDENCE_WEIGHTS = {
     INDICATOR_MATCH: 0.3,
     CODE_INDICATOR_MATCH: 0.4,
     FILE_INDICATOR_MATCH: 0.3,
     SELF_REFERENCE_MATCH: 0.3,
     FALLBACK_THRESHOLD: 0.8,
     FALLBACK_CONFIDENCE: 0.8
   };
   ```

2. **Security Considerations**

   **Finding**: Cache key generation uses simple lowercase transformation
   - Line 66: `const cacheKey = request.trim().toLowerCase();`
   
   **Risk**: Potential cache poisoning with Unicode homoglyphs
   
   **Recommendation**: Implement secure cache key generation
   ```javascript
   function generateSecureCacheKey(request) {
     const normalized = request.trim().toLowerCase();
     const hash = crypto.createHash('sha256');
     hash.update(normalized);
     return hash.digest('hex').substring(0, 16);
   }
   ```

3. **Maintainability Improvements**

   **Finding**: Category processing order is hard-coded
   - Line 136: `const categoryOrder = ['search', 'code', 'meta', 'complex', 'simple'];`
   
   **Recommendation**: Make configurable for easier tuning
   ```javascript
   const DEFAULT_CATEGORY_PRIORITY = ['search', 'code', 'meta', 'complex', 'simple'];
   
   function classifyRequest(request, options = {}) {
     const categoryOrder = options.categoryPriority || DEFAULT_CATEGORY_PRIORITY;
     // ... rest of implementation
   }
   ```

4. **Telemetry Enhancement**

   **Finding**: Telemetry logs grow unbounded in memory
   - Line 288: `telemetryLog.push(entry);`
   
   **Recommendation**: Implement circular buffer or persistent storage
   ```javascript
   class TelemetryLogger {
     constructor(maxEntries = 1000) {
       this.entries = [];
       this.maxEntries = maxEntries;
     }
     
     log(event, data) {
       if (this.entries.length >= this.maxEntries) {
         this.entries.shift(); // Remove oldest
       }
       this.entries.push({ timestamp: new Date().toISOString(), event, data });
     }
   }
   ```

5. **Pattern Matching Optimization**

   **Finding**: Regex patterns compiled on every request
   - Lines 148-153: Pattern test inside forEach loop
   
   **Recommendation**: Pre-compile regex patterns
   ```javascript
   const COMPILED_PATTERNS = {
     simple: {
       patterns: [/what is/i, /how do/i, /explain/i, /define/i].map(p => ({
         regex: p,
         test: (text) => p.test(text)
       }))
     }
     // ... other categories
   };
   ```
