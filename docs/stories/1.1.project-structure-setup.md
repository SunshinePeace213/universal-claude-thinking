### Project Structure
Based on the unified project structure, the following directories and files need to be created:
[Source: architecture/unified-project-structure.md]
```
universal-thinking-claude/
├── .github/                    # CI/CD workflows
│   └── workflows/
│       ├── test.yaml           # Module validation tests
│       └── security.yaml       # Security scanning
├── .claude/                    # Claude Code modules
│   ├── thinking-modules/       # Protocol modules
│   ├── cognitive-tools/        # Thinking operations
│   └── context-fields/         # (from AC, not in architecture)
├── scripts/                    # Utility scripts
├── tests/                      # Test suites
│   ├── unit/
│   ├── integration/
│   └── performance/
├── docs/                       # Documentation
├── .env.example               # Environment template
├── package.json               # Node.js config
└── README.md                  # Project overview
```

### Tech Stack Requirements
[Source: architecture/tech-stack.md]
- **Context Language**: Markdown (for module definition)
- **Configuration**: YAML 1.2 (for module metadata)
- **Version Control**: Git 2.x
- **Package Management**: Node.js with npm

### Coding Standards
[Source: architecture/coding-standards.md]
- Use kebab-case for module files and config files
- All paths must be validated and sanitized
- Follow security best practices for directory creation

### Git Ignore Patterns
Standard patterns for Claude Code projects should include:
- node_modules/
- .env (but not .env.example)
- *.log
- .DS_Store
- coverage/
- dist/
- build/

### Testing
[Source: architecture/testing-strategy.md#test-organization]
Test directory structure:
```
tests/
├── unit/
├── integration/
└── performance/
```

No specific testing requirements for this setup story, but the structure must support future test implementation.

### Technical Constraints
- All directories must be created with appropriate permissions for Claude Code to read/write
- The project must follow the exact structure defined in the architecture
- Git repository initialization should check if already initialized to avoid errors
